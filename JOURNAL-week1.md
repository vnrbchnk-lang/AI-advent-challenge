### День 3 — Разные способы рассуждения
- Статус: готово (код работает, 4 разных ответа получены)
- Файлы: `week1/day3/day3.py`
- Задача: «Как попить воду из кружки, у которой нет дна, а верхушка запаяна?» (латеральная загадка).
- Что сделано: один `ask()`-хелпер, та же модель `gpt-4o-mini`. 4 способа: (1) прямой; (2) «решай пошагово»; (3) модель сама составляет промпт, затем второй вызов решает по нему (2 запроса); (4) группа экспертов в system (аналитик/инженер/критик) + строка «Итог:». Печатает все 4 для сравнения.
- Результат сравнения: ответы заметно отличаются по стилю. Прямой — слабейший («выпить невозможно»). Пошагово/самопромпт — креативнее (трубочка, наклон, просверлить), но переусложнено. Эксперты — самый структурный (3 взгляда + итог). Изящный ответ «перевернуть кружку» не нашёл ни один метод — хороший материал для видео.
- Примечание: `sys.stdout.reconfigure("utf-8")` уже в коде; на Windows-консоли при показе всё равно полезно `$env:PYTHONIOENCODING="utf-8"`.

### День 4 — Температура
- Статус: готово (код работает)
- Файлы: `week1/day4/day4.py`
- Запрос (одна задача, покрывает все 3 критерия): «Сочини двустишие (две строки) про осень. Строки должны рифмоваться.»
- Что сделано: один `ask(temperature)`-хелпер, модель `gpt-5.2`. Одна задача × `temperature` 0 / 0.7 / 1.2. Рифма — проверяемое ограничение: ТОЧНОСТЬ = соблюдена ли рифма; КРЕАТИВНОСТЬ = образность; РАЗНООБРАЗИЕ = насколько разные варианты. На высокой temp рифма/смысл могут рассыпаться — наглядный trade-off для видео.
- Проверка API: gpt-5.2 на ProxyAPI **поддерживает** кастомную temperature (0/0.7/1.2 проходят; значение >2 отклоняется ошибкой `decimal_above_max_value` — реальная валидация, не заглушка). Внимание: **gpt-5.5 temperature НЕ поддерживает** — только дефолт =1 (`Unsupported value: 'temperature'... Only the default (1) value is supported`), поэтому для этого задания взят gpt-5.2.
- Выводы по применению: temp=0 — факты/код/классификация/повторяемость; temp=0.7 — ассистент, тексты, объяснения (баланс); temp=1.2 — брейншторм, варианты названий/идей.
- Примечание: ранние варианты запроса (название+тактика баскет-команды; пара креатив+факт) заменены на одну задачу с рифмой — она в одном ответе показывает все 3 критерия, коротко и наглядно.
- Примечание: `sys.stdout.reconfigure("utf-8")` в коде; на Windows-консоли при показе полезно `$env:PYTHONIOENCODING="utf-8"`.

### День 5 — Версии моделей
- Статус: готово (код работает, разница моделей видна)
- Файлы: `week1/day5/day5.py`
- Запрос (ловушка на внимательность): «У Салли 3 брата. У каждого из её братьев есть 2 сестры. Сколько сестёр у Салли?» — в семье 2 сестры (Салли + ещё одна), поэтому у каждого брата ровно 2 сестры, а у самой Салли сестёр = 1. Слабая модель тупо умножает/складывает; сильная рассуждает.
- Что сделано: один `ask(model, prompt)`-хелпер. Один запрос × 3 модели (слабая/средняя/сильная): `gpt-3.5-turbo` / `gpt-4o` / `gpt-5.5`. Замеряет **время** (`time.perf_counter`), **токены** (`usage`: вход/выход/всего). Печатает по каждой модели: время, токены, цена/1М (вход/выход), ответ. Цену за конкретный запрос не считаем — её пользователь смотрит на сайте ProxyAPI.
- Токены: **вход** (prompt) = отправленные модели сообщения (платишь за чтение); **выход** (completion) = сгенерированный ответ (платишь за генерацию, обычно дороже входа в разы).
- Цены (за 1М токенов, всё в USD единой валютой): gpt-3.5-turbo $0.50/$1.50, gpt-4o $2.50/$10.00 — официальный прайс OpenAI; gpt-5.5 — новее cutoff, цены OpenAI нет → прайс ProxyAPI 1520₽/9100₽, конвертирован в $ через константу `RUB_PER_USD = 90` (поправить при другом курсе).
- Замечания по API: `-pro`-модели (o1-pro, gpt-5-pro, gpt-5.5-pro) в `/chat/completions` дают **404** — они только на `/responses`, поэтому сильная = `gpt-5.5` (не pro). Имена моделей сверены с живым списком `GET /openai/v1/models`.
- Результат сравнения (фактический прогон): 3.5-turbo — заученная многоходовка, подвох не заметила (❌), ~2.0 c, 270 токенов; gpt-4o — заметила подвох, «один рейс» (✅), ~1.4 c, 155 токенов; gpt-5.5 — заметила подвох, ответ чище всех (✅), ~3.4 c, 143 токена. Вывод: сильнее модель → читает условие внимательнее (качество), тратит меньше токенов на суть, но медленнее и дороже за токен.
- Примечание: `sys.stdout.reconfigure("utf-8")` в коде → запуск просто `python week1\day5\day5.py`, env-переменная не нужна.
