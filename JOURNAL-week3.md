### День 11 — Модель памяти ассистента (memory layers)
- Статус: код готов, ревью пройдено; живой прогон — за пользователем (правило 11, экономия токенов)
- Файлы: `week3/assistant/memory.py`, `week3/assistant/cli11.py`, `week3/pyproject.toml`
- Задание: описать и реализовать модель памяти ассистента, минимум 3 типа — краткосрочная (диалог), рабочая (данные текущей задачи), долговременная (профиль/решения/знания); хранить отдельно; явно выбирать, что и куда сохраняется; проверить, какие данные попадают в каждый слой и как это влияет на ответы.
- Архитектура: `MemoryLayers` (memory.py) — три слоя, каждый в **отдельном файле** `week3/assistant/store/`: `short_term.json` (список реплик), `working.json` (KV-данные задачи), `long_term.json` (KV-профиль/решения). «Хранятся отдельно» — буквально три файла (нагляднее для видео). `build_messages()` — prompt builder: `[system] + [system: долговременная] + [system: рабочая] + последние N реплик`. Слои подмешиваются в запрос **явными флагами** `use_long_term`/`use_working` — можно включать/выключать, чтобы показать влияние на ответ.
- Явный выбор «что куда» — через слэш-команды CLI (а не авто-роутер): `/remember название = содержимое` → долговременная; `/task название = содержимое` → рабочая; обычное сообщение (без слэша) → краткосрочная (диалог). Память — KV-заметки (название=содержимое); без «=» название генерится само.
- Набор команд намеренно минимальный (6, после упрощения по просьбе пользователя — раньше было 13, «невозможно разобраться»): `/remember`, `/task`, `/memory`, `/demo`, `/reset`, `/help`. Убраны `/forget`, `/newtask`, `/clear`, `/prompt`, `/longterm on|off`, `/working on|off` — гранулярная чистка и тумблеры слоёв перегружали, а влияние памяти и так показывает `/demo`. Методы чистки остались в `memory.py` (пригодятся в следующих днях недели), просто не выведены в CLI.
- Проверка «что в каждом слое»: команда `/memory` — таблица из 3 слоёв с колонкой «что хранит».
- Проверка «как влияет на ответы»: команда **`/demo`** внутри CLI — A/B на одном вопросе «набросай сервис авторизации». Профиль в долговременной (стек Kotlin/MVI, запрет Python). Слева запрос БЕЗ долговременной → ходовой вариант (Python); справа С долговременной → Kotlin/MVI. Демо гоняет на временных сообщениях, живую память не трогает.
- Модель: **`gpt-4.1`** (смена с gpt-5.2 ради экономии, решение пользователя — выход 2062₽/1М против 4245₽ у gpt-5.2, вдвое дешевле; цена сверена на https://proxyapi.ru/pricing/list). Учёт токенов и цены (₽, прайс ProxyAPI) в подвале каждого хода.
- CLI: красивый интерфейс через `rich` (правило 12) — панели, цветные таблицы слоёв, спиннер «думает», баннер на старте. Слэш-команды как в Claude Code: ввод через `prompt_toolkit`, при наборе `/` всплывает список команд с описанием (автодополнение). `/help` — подробная справка с объяснением KV-памяти.
- Глобальная команда: **`agent11`** (`pip install -e week3`), пакет `assistant`, console-script → `assistant.cli11:main`.
- Сверх ТЗ: глобальная команда + rich-интерфейс; флаги подмешивания слоёв и команда `prompt` (видно сборку запроса); персистентность всех трёх слоёв между запусками; раздельный учёт токенов/цены. Заявка на неделю: это первый кирпич растущего stateful-агента (дальше — профиль/state machine/инварианты).
- Запуск (пользователь): `pip install -e week3` → `agent11`, демо — команда `demo` внутри CLI. `store/*.json` в .gitignore.
