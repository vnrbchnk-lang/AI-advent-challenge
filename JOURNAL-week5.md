## Решения недели (зафиксировано 2026-07-04)

- **Эмбеддинги — локально, Ollama + `bge-m3`** (мультиязычная, 1024-dim, русский держит).
  Выбор пользователя: privacy — база знаний не покидает машину; бесплатно; путь из лекции.
  `nomic-embed-text` отвергнута (слаба в русском).
- **Корпус — база хакатона `C:\Alaba`** («Личная культура», ККИ про наследие России):
  `.memory-bank/*.md` (правила игры, архитектура, конвенции, транскрипты встреч) + `raw/*.txt`
  (ТЗ, транскрипты) + `raw/USM.pdf` (pypdf) + README. Итог: 20 документов, ~201К символов
  (~110 страниц), три формата — «README / статьи / pdf → текст» из ТЗ дня 21 закрыты.
  **Контент в репозиторий не коммитится** (данные заказчика): `rag/docs/` и `rag/store/`
  в .gitignore, сборка читает исходники из `C:\Alaba`.
- **Индекс — JSON + numpy**: векторы нормализованы при сохранении, косинус = скалярное
  произведение, брутфорс по ~150-170 чанкам мгновенный. FAISS — оверкилл, содержимое JSON
  можно открыть и показать.
- Генерация — как всю дорогу: ProxyAPI, `gpt-4.1` (ответы), `gpt-4o-mini` (rewrite/rerank/
  экстракция памяти). CLI — rich + prompt_toolkit, navy, без эмодзи.

### День 21 — Индексация документов
- Статус: в работе
- Файлы: `week5/rag/corpus.py`, `chunking.py`, `embedder.py`, `index.py`, `cli.py`
- Задание: пайплайн индексации: chunking, эмбеддинги, сохранение индекса; метаданные к чанкам
  (source, title, section, chunk_id); минимум 2 стратегии chunking и их сравнение.
- Что сделано: `/fetch` собирает корпус (md как есть, txt заворачивается, pdf через pypdf);
  две стратегии: **fixed** (1600 символов, overlap 200, по границам предложений) и **structural**
  (по заголовкам markdown, путь заголовков в `section`, длинные секции дорезаются, короткие
  склеиваются); `/index` строит оба индекса; `/search` показывает топ-5 из обоих рядом.
  Метаданные чанка: `chunk_id, strategy, source, group, title, section, n_chars`.
- Сравнение стратегий: fixed — 149 чанков (avg 1473, рваных концов 23), structural — 164
  (avg 1233, рваных 14). На структурных md выигрывает structural (чанк = смысловая секция,
  «Травма и смерть карты» целиком), на сырых транскриптах разница стирается — там и структуры нет.
- Сверх ТЗ: нормализация векторов на записи; счётчик «рваных концов» как метрика качества чанкинга.

### День 22 — Первый RAG-запрос
- Статус: в работе
- Файлы: `week5/rag/rag.py`, `evalset.py`
- Задание: вопрос → поиск чанков → объединение с вопросом → LLM; сравнить с/без RAG;
  10 контрольных вопросов с ожиданием и ожидаемыми источниками.
- Что сделано: `/ask` (RAG: топ-K чанков с метаданными в промпт, «отвечай только по контексту»)
  vs `/raw` (без RAG); `/eval` — 10 вопросов по базе хакатона из `evalset.py`
  (вопрос/ожидание/ожидаемые источники), автопроверка «ожидаемый источник среди использованных».
- Сверх ТЗ: автоматическая сверка источников кодом, не глазами.

### День 23 — Реранкинг и фильтрация
- Статус: в работе
- Файлы: `week5/rag/retrieval.py`
- Задание: второй этап после поиска (reranker/фильтр), порог отсечения, топ-K до и после,
  сравнение качества, query rewrite.
- Что сделано: конвейер `rewrite (gpt-4o-mini, опц.) → векторный поиск topN=20 → порог косинуса
  (0.5) → LLM-реранк (gpt-4o-mini, оценка 0-10, отсечка <4, опц.) → topK=5 в промпт`.
  `/mode` — все ручки (strategy/rewrite/topn/threshold/rerank/rerankmin/topk);
  `/compare23 <q>` — сырой топ-12 и судьба каждого чанка (в промпт / срезан порогом / срезан
  реранком). Локальный cross-encoder не ставили: torch на Windows тяжёлый, LLM-реранк дешёвый.

### День 24 — Цитаты, источники и анти-галлюцинации
- Статус: в работе
- Файлы: `week5/rag/rag.py` (формат + валидация)
- Задание: обязательные ответ + источники (source, section, chunk_id) + цитаты; проверить на
  10 вопросах; при слабой релевантности — «не знаю» и просьба уточнить.
- Что сделано: ответ строго JSON `{answer, sources, quotes}`; каждая цитата **валидируется
  кодом** — подстрока ли она чанка (пометка «дословно»/«НЕ НАЙДЕНА»); «не знаю» — детерминированный
  код-гейт: если после фильтров чанков нет, LLM для ответа вообще не вызывается. `/check` —
  чек-таблица 10 вопросов (источники/цитаты/дословность + колонка ручной сверки смысла для видео);
  `/demo24` — обычный вопрос + вопрос мимо базы («столица Австралии») → отказ.
- Сверх ТЗ: валидация цитат подстрокой; отказ без вызова LLM (в духе код-гейтов дня 15).

### День 25 — Мини-чат с RAG + памятью
- Статус: в работе
- Файлы: `week5/rag/taskmemory.py`, `cli.py` (`/chat`)
- Задание: чат с историей; на каждый вопрос — RAG-поиск; ответы с учётом контекста; всегда
  источники; память задачи (что уточнено, ограничения/термины, цель диалога); 2 длинных сценария.
- Что сделано: `/chat` — вложенный prompt-loop (паттерн week3): история диалога (окно 8 сообщений
  в промпт), query rewrite с учётом истории (реюз дня 23 — «а что с редкостью?» разворачивается
  в самодостаточный запрос), RAG-конвейер дней 22-24 → источники всегда; память задачи
  `{goal, clarified, constraints, glossary}` обновляется экстрактором gpt-4o-mini после каждого
  обмена, живёт в `store/chat-default.json` (переживает перезапуск); `/state`, `/reset`;
  bottom_toolbar показывает цель и число фактов.
- Сверх ТЗ: rewrite вопроса с учётом истории (склейка дней 23+25); resume чата между запусками.
