## 3. Инструменты (стек проекта)

| Роль | Выбор | Детали |
|------|-------|--------|

| LLM API (для заданий) | **ProxyAPI.ru** | Российский посредник, оплата рублями с карты РФ. Один ключ → DeepSeek / OpenAI / Gemini / Anthropic. OpenAI-совместимый формат. |
| Язык/среда | (определить) | Рекомендуется Python или Node.js — минимальный код для REST-запросов. |
| VPS/хостинг | (пока не требуется) | Понадобится позже по программе. Варианты: timeweb.com, beget.com/ru/vps, hosting-vds.com. |

### Harness ≠ API (не путать)
- **Claude Code (harness)** — инструмент, который помогает *писать и запускать код*. Работает на подписке Claude.
- **ProxyAPI/DeepSeek (API из задания)** — облачная LLM, к которой *обращается написанный код* по REST. Именно это требуют задания («облачная модель, доступная через REST»).

### Доступ к ProxyAPI
- Регистрация и ключ: https://console.proxyapi.ru (нужен только email, без иностранного телефона).
- Оплата: рублями с карты любого банка РФ.
- Авторизация во всех запросах: заголовок `Authorization: Bearer <КЛЮЧ>`.
- Базовый адрес: `https://api.proxyapi.ru`, дальше идентификатор провайдера:
  - OpenAI (подтверждено докой): `https://api.proxyapi.ru/openai/v1` → `/chat/completions`
  - DeepSeek: путь вида `https://api.proxyapi.ru/deepseek` — **ТОЧНЫЙ путь подтвердить в консоли/доке ProxyAPI после регистрации** и зафиксировать здесь.
- Ключ хранить в переменной окружения (`PROXYAPI_KEY`), НЕ коммитить в код/git.

### Запуск Claude Code в терминале (памятка)
1. Установить Node.js LTS (https://nodejs.org).
2. `npm install -g @anthropic-ai/claude-code`
3. `cd C:\ADVENT` → `claude` → залогиниться своим аккаунтом Claude.
4. Удобно: открыть `C:\ADVENT` в VS Code, встроенный терминал (`Ctrl+~`), запустить `claude`.

## 4. ОГРАНИЧЕНИЯ ДЛЯ АГЕНТА (строго соблюдать)

Эти правила обязательны при работе над любым заданием челленджа:

1. **Делать ровно то, что требует задание дня — не больше.** Не добавлять фич, абстракций,
   «улучшений», фреймворков, конфигов, тестов, которые задание не просит.
2. **Не выдумывать.** Если задание чего-то не уточняет — спросить пользователя, а не
   достраивать предположения. Не изобретать несуществующие требования.
3. **Минимализм кода.** Задания первых дней решаются в десятки строк. Держать решение
   максимально простым и читаемым. Никакого оверинжиниринга.
4. **Точность фактов про API.** Не выдумывать эндпоинты, имена моделей, параметры. Если
   не уверен — проверить в актуальной документации или спросить, а не угадывать.
5. **Не выходить за рамки текущего дня.** Не начинать задания будущих дней заранее без
   явной просьбы пользователя.
6. **Соответствие формату сдачи.** Помнить, что результат = Видео + Код. Код должен быть
   таким, чтобы пользователь мог его легко показать и объяснить на видео.
7. **Безопасность.** API-ключи только в переменных окружения, не в коде и не в репозитории.
8. **Сверяться с формулировкой.** Перед сдачей сверять результат с дословной формулировкой
   задания (см. файл `первые2днячелленджа.txt` и последующие).
9. **Журналировать.** После выполнения задания дня дополнять раздел «Журнал по дням» ниже:
   что сделано, какие файлы созданы, ссылка на код, статус сдачи.

## 5. Задания (формулировки)

Полные тексты заданий первых двух дней — в файле `первые2днячелленджа.txt`.

### День 1. Первый запрос к LLM через API
Написать минимальный код, который:
- отправляет запрос в LLM через API;
- получает ответ;
- выводит его в консоль или простой интерфейс (CLI / Web).
**Результат:** код, отправляющий запрос в LLM через API и получающий ответ.
**Формат:** Видео + Код.

### День 2. Формат ответа
Отправить тот же запрос, но:
- добавить явное описание формата ответа;
- добавить ограничение на длину ответа;
- добавить условие завершения ответа (stop sequence или явная инструкция).
Сравнить ответы: без ограничений vs с ограничениями.
**Результат:** один и тот же запрос с разным уровнем контроля ответа через API.
**Формат:** Видео + Код.
